# CI/CD Documentation

## Overview

Этот проект использует современные CI/CD практики для автоматизации тестирования, сборки и развертывания.

## GitHub Actions

### Workflow: `.github/workflows/ci.yml`

#### Запускается при:
- Push в ветки `main` и `develop`
- Pull Request в ветку `main`

#### Этапы:

1. **Test**
   - Запуск unit-тестов с покрытием кода
   - Загрузка отчетов в Codecov
   - Генерация HTML отчета о покрытии

2. **Lint**
   - Запуск golangci-lint
   - Проверка нашим линтером на тестовых данных

3. **Build**
   - Сборка для всех платформ (Linux, Windows, macOS)
   - Поддержка архитектур amd64 и arm64
   - Оптимизация бинарных файлов

4. **Security**
   - Сканирование безопасности Gosec
   - Загрузка SARIF отчетов

5. **Benchmark**
   - Запуск бенчмарков
   - Отслеживание производительности

6. **Docker** (только для main ветки)
   - Сборка и публикация Docker образа
   - Мультиплатформенная сборка

7. **Release** (только для тегов)
   - Создание GitHub Release
   - Публикация артефактов

#### Переменные окружения:
- `GO_VERSION`: Версия Go (1.22)
- `GITHUB_TOKEN`: Для создания релизов
- `CODECOV_TOKEN`: Для загрузки покрытия

## GitLab CI/CD

### Конфигурация: `.gitlab-ci.yml`

#### Стадии:

1. **test**
   - `test:unit`: Unit-тесты с покрытием
   - `test:race`: Тесты с race condition
   - `test:memory`: Тесты с профилированием памяти
   - `test:benchmark`: Бенчмарки

2. **lint**
   - `lint:golangci`: Запуск golangci-lint
   - `lint:our-linter`: Проверка нашим линтером
   - `lint:format`: Проверка форматирования кода
   - `lint:vulnerability`: Проверка уязвимостей зависимостей

3. **build**
   - `build:linux`: Сборка для Linux
   - `build:darwin`: Сборка для macOS
   - `build:windows`: Сборка для Windows
   - `build:docker`: Сборка Docker образа

4. **security**
   - `security:gosec`: Сканирование безопасности
   - `security:sast`: SAST анализ

5. **deploy**
   - `deploy:release`: Создание релиза (для тегов)
   - `deploy:registry`: Публикация в registry

#### Кэширование:
- Go модули: `$GO_MOD_CACHE_DIR`
- Сборка: `$GO_BUILD_CACHE_DIR`

#### Артефакты:
- Отчеты о покрытии кода
- Бинарные файлы для всех платформ
- Результаты бенчмарков
- Отчеты безопасности

## Docker

### Dockerfile

Многостадийный Dockerfile:
- **Stage 1 (builder)**: Сборка бинарного файла
- **Stage 2 (final)**: Минимальный runtime образ

#### Особенности:
- Основан на Alpine Linux
- Статически скомпилированный бинарник
- Health check для мониторинга
- Оптимизированный размер образа

### Сборка и публикация:

```bash
# Локальная сборка
docker build -t loglinter .

# Сборка с тегом
docker build -t loglinter:1.0.0 .

# Запуск
docker run --rm -v $(pwd):/app loglinter ./app
```

## Релизы

### Автоматические релизы:

1. **GitHub Releases**:
   - Создаются при пуше тега
   - Включают бинарные файлы для всех платформ
   - Автоматическая генерация checksum

2. **GitLab Releases**:
   - Создаются при пуше тега
   - Включают все артефакты сборки
   - Генерация checksums.txt

### Версионирование:

- Используйте семантическое версионирование (SemVer)
- Формат: `vX.Y.Z`
- Пример: `v1.0.0`, `v1.1.0`, `v1.0.1`

## Локальная разработка

### Пред-commit проверки:

```bash
# Запуск всех проверок
make pre-commit

# Или по отдельности
make test
make lint
make build
```

### Локальный CI/CD:

```bash
# GitHub Actions
act -j test

# GitLab CI (требует GitLab Runner)
gitlab-ci-local
```

## Траблшутинг

### Частые проблемы:

1. **Кэш Go модулей**:
   ```bash
   go clean -modcache
   go mod download
   ```

2. **Проблемы с Docker**:
   ```bash
   docker system prune -a
   docker build --no-cache
   ```

3. **Проблемы с CI**:
   - Проверьте переменные окружения
   - Убедитесь что все секреты настроены
   - Проверьте права доступа

### Логирование:

- CI/CD логи доступны в интерфейсе платформы
- Локальные логи сохраняются в артефактах
- Используйте `-v` флаг для подробного вывода

## Безопасность

### Сканирование:
- Gosec для статического анализа
- Govulncheck для уязвимостей зависимостей
- SAST для комплексного анализа

### Секреты:
- Все секреты хранятся в переменных окружения
- Используются CI/CD секреты платформы
- Никаких секретов в коде
