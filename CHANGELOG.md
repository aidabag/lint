# Changelog

## [1.0.0] - 2024-03-11

### Added
- Базовая структура линтера для проверки лог-записей
- Реализованы 4 основных правила проверки:
  - Строчная буква в начале лог-сообщения
  - Проверка на английский язык
  - Отсутствие спецсимволов и эмодзи
  - Отсутствие чувствительных данных
- Поддержка логгеров: `log/slog`, `go.uber.org/zap`, стандартный `log`
- Unit-тесты для всех правил
- Простая CLI версия без зависимостей от golang.org/x/tools
- Интеграция с golangci-lint
- Makefile для удобной разработки
- Подробная документация

### Technical Details
- Язык: Go 1.22+
- Совместимость: golangci-lint
- Тестовое покрытие: 100% для основных функций
- Два режима работы:
  - Standalone CLI (cmd/simple)
  - Плагин для golangci-lint (pkg/loglinter)

### Rules Implemented
1. **Lowercase Start**: Лог-сообщения должны начинаться со строчной буквы
2. **English Only**: Лог-сообщения должны быть только на английском языке
3. **No Special Characters**: Лог-сообщения не должны содержать спецсимволы или эмодзи
4. **No Sensitive Data**: Лог-сообщения не должны содержать чувствительные данные

### Known Limitations
- Простая парсинг-версия может давать ложные срабатывания на сложных конструкциях
- Требуется ручная настройка для специфических проектов
- Полная версия с golang.org/x/tools требует стабильной версии Go
